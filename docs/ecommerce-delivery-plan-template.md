# E-commerce Public Demo Plan Boundary

This file no longer contains a customer delivery plan template.

The public repository only keeps a high-level demo validation outline so developers can understand the mock ecommerce pack without receiving commercial implementation playbooks.

## Public Demo Flow

1. Review mock Pack Config, DSL, policies, templates, and adapter contracts.
2. Run local demo cases with mock adapters.
3. Inspect trace/audit output.
4. Run certification checks.
5. Keep any production planning in private implementation repositories.

## Not Included

- project schedule
- customer responsibility matrix
- production rollout plan
- UAT sign-off plan
- private SOP
- commercial delivery checklist

This boundary keeps public Yutra focused on open-source core and mock/demo examples.
