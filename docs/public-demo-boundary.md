# Public Demo Boundary

Yutra's public repository includes the open-source core and mock/demo examples for developer learning, local validation, trace/audit inspection, and certification.

It does not include commercial or private implementation assets.

## Public Repository Includes

- execution standard and reference runtime
- DSL, canonical IR, trace, audit, and certification examples
- Skill, archetype, Pack Config, and Rule Compiler core packages
- mock/demo scenario packs
- local demo inputs
- non-production adapter contracts
- public documentation for architecture and boundaries

## Public Repository Excludes

- pricing, quotation, or proposal material
- production UAT or rollout playbooks
- customer-specific SOP, templates, configs, or policies
- real customer endpoints
- real customer data
- credentials, secrets, API keys, or private tokens
- production adapter mapping
- hosted dashboards or customer operations consoles

## Demo Asset Rule

Examples in this repository are for developer learning and local validation only. They should be treated as mock/demo assets, not customer delivery assets.

Any customer-specific implementation material should live in private repositories or private deployment environments.
